# MatchBox WASM + JS Template

This is a modern, simplified template for building WebAssembly-powered applications using **BoxLang** and **Alpine.js**.

## Features

- **Single-File Logic**: All application state and business logic is contained in `app.bxs`.
- **Alpine.js Components**: Uses the latest MatchBox conventions to register Alpine components directly from BoxLang.
- **Tailwind CSS**: Modern, utility-first styling for a polished look.
- **WASM Performance**: BoxLang code is compiled to WASM for near-native performance in the browser.
- **Zero Boilerplate**: No complex framework wiring; BoxLang owns the component registration.

## Project Structure

- `app.bxs`: The single source of truth for your application state and logic.
- `index.html`: The host page, styled with Tailwind CSS and driven by Alpine.js.
- `app.js` / `app.wasm`: The compiled assets (generated by MatchBox).

## Getting Started

### 1. Install MatchBox
Ensure you have the latest MatchBox CLI installed:
```bash
curl -sSL https://raw.githubusercontent.com/ortus-boxlang/matchbox/master/install/install.sh | bash
```

### 2. Build the Project
Compile your BoxLang code into a JavaScript/WASM bundle:
```bash
matchbox --target js app.bxs
```
This produces `app.js` and `app.wasm`.

### 3. Run the Development Server
Serve the project using any local web server:
```bash
matchbox --serve --port 8080
```
Then open `http://localhost:8080` in your browser.

## How it Works

1. **Direct Registration**: The BoxLang script uses `js.Alpine.data()` to register an Alpine component named `bxTodoApp`.
2. **Host Mutation**: BoxLang functions receive the Alpine `$data` proxy, allowing them to update reactive UI state directly from WASM.
3. **Automatic Conversion**: BoxLang Structs and Arrays are automatically converted into reactive JavaScript objects that Alpine can track.
