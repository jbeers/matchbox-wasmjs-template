# MatchBox WASM + JS Template

This is a modern template for building WebAssembly-powered applications using **BoxLang** and **Alpine.js**.

## Features

- **BoxLang Logic**: All application state and business logic is written in BoxLang.
- **Alpine.js UI**: A thin, reactive UI layer using Alpine.js, driven directly by BoxLang.
- **WASM Performance**: BoxLang code is compiled to WASM for near-native performance in the browser.
- **Zero Boilerplate**: Uses the latest MatchBox conventions for direct Alpine component registration.
- **Tailwind CSS**: Modern, utility-first styling.

## Project Structure

- `app.bxs`: The main entry point for your BoxLang application logic.
- `models/`: BoxLang classes representing your domain data.
- `services/`: BoxLang classes handling business logic and state management.
- `index.html`: The host page, using Alpine.js and Tailwind CSS.
- `app.js` / `app.wasm`: The compiled assets (generated by MatchBox).

## Getting Started

### 1. Install MatchBox
Ensure you have the latest MatchBox CLI installed:
```bash
curl -sSL https://raw.githubusercontent.com/ortus-boxlang/matchbox/master/install/install.sh | bash
```

### 2. Build the Project
Compile your BoxLang code into a JavaScript/WASM bundle:
```bash
matchbox --target js app.bxs
```
This produces `app.js` and `app.wasm`.

### 3. Run the Development Server
Serve the project using any local web server:
```bash
# Using Python
python3 -m http.server 8000

# Using Node.js
npx serve .
```
Then open `http://localhost:8000` in your browser.

## How it Works

1. **Direct Registration**: The BoxLang script uses `js.Alpine.data()` to register an Alpine component named `bxTodoApp`.
2. **Host Mutation**: BoxLang functions receive the Alpine `$data` proxy, allowing them to update reactive UI state directly from WASM.
3. **Encapsulation**: Domain logic is separated into BoxLang Classes and Services, while the UI remains focused on presentation.
